### Exact build/submit commands

```bash
# one-time setup, after account creation
npx eas login
npx eas init                                         # writes extra.eas.projectId
npx eas credentials -p ios                           # configure iOS signing (interactive)
npx eas credentials -p android                       # configure Android keystore (interactive)

# preview / TestFlight / internal-testing builds (optional but recommended)
npx eas build -p ios --profile preview
npx eas build -p android --profile preview

# production builds
npx eas build -p ios --profile production
npx eas build -p android --profile production

# submit
npx eas submit -p ios --profile production --latest
npx eas submit -p android --profile production --latest
```

### Local prebuild / dry-run

To inspect what a production build would package without running EAS servers:

```bash
npx expo prebuild --clean
# Now ios/ and android/ folders are generated; the JS bundle can be exported with:
npx expo export --platform ios
npx expo export --platform android
```

The `prebuild` step is gitignored (ios/ + android/ are in `.gitignore`) — EAS regenerates them on each build, so checking them in is unnecessary.

**Production-build attempt:** not run from this environment — no EAS account credentials available. The config is in place; a single `npx eas build -p ios --profile production` (after `eas login` + `eas init`) is the entire production-build command.

## Precise remaining human steps — to hit "Submit for Review"

### Apple App Store
1. **Pay** $99/yr Apple Developer Program enrollment (apple.com/developer).
2. In **App Store Connect**:
   - Create a new App: name **Lattice — 4D Tic-Tac-Toe**, bundle ID `com.raban.lattttice`, primary language English.
   - Note the **App ID** (the numeric `ascAppId`) and your **Team ID**.
   - In `eas.json`, replace `REPLACE_WITH_APPLE_ID_EMAIL`, `REPLACE_WITH_APPSTORECONNECT_APP_ID`, `REPLACE_WITH_APPLE_TEAM_ID`.
3. **Locally** (or in CI):
   ```bash
   npx eas login
   npx eas init                                  # writes extra.eas.projectId
   npx eas credentials -p ios                    # let EAS generate distribution cert + provisioning
   npx eas build -p ios --profile production
   npx eas submit -p ios --profile production --latest
   ```
4. In App Store Connect:
   - Upload **icon**, **screenshots** (6.7" set required; 5.5" set required for older devices), **app preview** video (optional).
   - Paste **description**, **subtitle**, **keywords**, **promotional text**, **support URL**, **marketing URL**, **privacy policy URL** (all from `STORE-LISTING.md`).
   - Fill the **App Privacy** section: every answer is **No** (no data collected).
   - Complete **Age Rating questionnaire**: every answer is **None / No** → expect 4+.
   - **Export Compliance:** already set in plist (`ITSAppUsesNonExemptEncryption: false`); App Store Connect will pre-fill.
5. Hit **Submit for Review**.

### Google Play
1. **Pay** $25 one-time Google Play Console registration (play.google.com/console).
2. In **Play Console**:
   - Create a new app: name **Lattice: 4D Tic-Tac-Toe**, default language English (United States), Game, Free.
   - Complete the **app content** declarations (privacy policy URL, ads = No, app access = no restrictions, content rating questionnaire, target audience, news app = No, COVID-19 contact tracing = No, data safety = no data collected).
3. **Service account for EAS Submit:**
   - In Google Cloud Console, create a service account with role "Service Account User" + grant it access in Play Console (Users and permissions → Invite new user → service-account email → grant "Release manager" role for this app).
   - Download the JSON key, save as `play-store-service-account.json` at the repo root (gitignored — already in `.gitignore` via the `.env*` and `*.jks` patterns; add `play-store-service-account.json` to `.gitignore` if not covered).
4. **Locally** (or in CI):
   ```bash
   npx eas login                                    # if not done
   npx eas credentials -p android                   # generate / upload keystore
   npx eas build -p android --profile production    # produces .aab
   npx eas submit -p android --profile production --latest
   ```
5. In Play Console:
   - Upload **app icon**, **feature graphic** (1024×500), **phone screenshots**, optional 7"/10" tablet screenshots.
   - Paste **short** + **full** description from `STORE-LISTING.md`.
   - Set the release as **Production** (or first push to **Internal testing** if you want a stage gate).
   - Confirm the **Data safety** form is complete.
6. Hit **Send for review** (Play Console wording — equivalent to App Store's "Submit for Review").
